API ENDPOINTS - SIMPLE OVERVIEW
================================
Base URL: /api
Auth: Send token in header -> Authorization: Bearer <token>

--------------------------------------------------
USER ENDPOINTS (/api/user)
--------------------------------------------------

1) POST /api/user/register
   Purpose: Creates a brand new user account.

   req.body =
   {
     email: string,
     phone: string,
     password: string,
     fullName: string,
     birthday: string
   }

   res =
   {
     success: boolean,
     message: string
   }

2) POST /api/user/login
   Purpose: Logs a user in and gives them a token to use for future requests.

   req.body =
   {
     email_or_phone: string,
     password: string
   }

   res =
   {
     success: boolean,
     message: string,
     token: string
   }

3) POST /api/user/logout
   Purpose: Logs the user out by cancelling their current token.

   req.headers =
   {
     Authorization: "Bearer <token>"
   }

   req.body = {}   (nothing needed)

   res =
   {
     success: boolean,
     message: string
   }

4) GET /api/user/me
   Purpose: Gets the profile info of the user who is currently logged in.

   req.headers =
   {
     Authorization: "Bearer <token>"
   }

   res =
   {
     success: boolean,
     message: string,
     user:
     {
       id: string,
       email: string,
       phone: string,
       fullName: string,
       birthday: string,
       createdAt: string,
       permission_level: number
     }
   }

5) PUT /api/user/me
   Purpose: Lets the logged-in user update their own account details.

   req.headers =
   {
     Authorization: "Bearer <token>"
   }

   req.body =
   {
     password?: string,
     email?: string,
     phone?: string,
     fullName?: string,
     birthday?: string
   }

   res =
   {
     success: boolean,
     message: string,
     user:
     {
       id: string,
       email: string,
       phone: string,
       fullName: string,
       birthday: string,
       createdAt: string,
       permission_level: number
     }
   }

PROFILE ENDPOINTS (/api/profile)
--------------------------------------------------
(All profile endpoints need: req.headers = { Authorization: "Bearer <token>" })
(A "profile" is like a Netflix-style profile - one user account can have several profiles, max 4)

6) POST /api/profile/
   Purpose: Creates a new profile under the logged-in user's account (max 4 profiles).

   req.body = {}   (nothing needed)

   res =
   {
     success: boolean,
     message: string,
     profiles:
     [
       { id: string, profileName: string, age: number, ImageName: string },
       ...
     ]
   }

7) PUT /api/profile/
   Purpose: Updates several profiles belonging to the user in one request.

   req.body =
   {
     updates:
     [
       { profileId: string, profileName?: string, age?: number, ImageName?: string },
       ...
     ]
   }

   res =
   {
     success: boolean,
     message: string,
     profiles:
     [
       { id: string, profileName: string, age: number, ImageName: string },
       ...
     ]
   }

8) GET /api/profile/
   Purpose: Gets the full list of all profiles that belong to the logged-in user.

   req.body = {}   (nothing needed)

   res =
   {
     success: boolean,
     message: string,
     profiles:
     [
       { id: string, profileName: string, age: number, ImageName: string },
       ...
     ]
   }

9) DELETE /api/profile/:profileId
   Purpose: Deletes one of the user's profiles (must always keep at least one).

   }

   res =
   {
     success: boolean,
     message: string,
     profiles:

       ...

   }

10) PUT /api/profile/:profileId
    Purpose: Updates the details of one profile (name, age, or picture).

11) GET /api/profile/:profileId
    Purpose: Gets the basic details of one specific profile.

    req.params =
    {

      success: boolean,

      {
        id: string,
        profileName: string,
        age: number,
        ImageName: string
      }
    }

12) GET /api/profile/:profileId/details
    Purpose: Gets the complete details of one profile, including watch history and liked content.

    req.params =
    {
      profileId: string
    }

    res =
    {
      success: boolean,
      profile:
      {
        id: string,
        profileName: string,
        age: number,
        ImageName: string,
        likedContentIds: [ contentId, ... ],
        lastWatched:
        [
          { _id: string, episode_id: string, content_id: string },
          ...
        ]
      }
    }

    Note: lastWatched has one entry per content the profile has ever watched (not one per episode) -
    each entry holds the specific episode last watched within that content.

13) POST /api/profile/:profileId/likes/:contentId
    Purpose: Adds or removes a "like" on a specific movie/show for a specific profile (toggle).

    req.params =
    {
      profileId: string,
      contentId: string
    }

    req.body = {}   (nothing needed)

    res =
    {
      success: boolean,
      message: string,
      liked: boolean,
      likedContentIds: [ contentId, ... ]
    }

14) POST /api/profile/:profileId/watch/:contentId
    Purpose: Records that a profile just watched a movie/show, with no specific episode given.
    Behavior: if this content already has a saved entry in the profile's watch history, resumes
    from that saved episode; otherwise starts from season 1, episode 1.

    req.params =
    {
      profileId: string,
      contentId: string
    }

    req.body = {}   (nothing needed)

    res =
    {
      success: boolean,
      message: string,
      episode:
      {
        id: string,
        contentId: string,
        seasonNumber: number,
        episodeNumber: number,
        title: string,
        videoUrl: string
      },
      lastWatched:
      [
        { _id: string, episode_id: string, content_id: string },
        ...
      ]
    }

15) POST /api/profile/:profileId/watch/:contentId/:episodeId
    Purpose: Same as above, but records a specific episode as watched instead of resuming/defaulting.

    req.params =
    {
      profileId: string,
      contentId: string,
      episodeId: string
    }

    req.body = {}   (nothing needed)

    res =
    {
      success: boolean,
      message: string,
      episode:
      {
        id: string,
        contentId: string,
        seasonNumber: number,
        episodeNumber: number,
        title: string,
        videoUrl: string
      },
      lastWatched:
      [
        { _id: string, episode_id: string, content_id: string },
        ...
      ]
    }

16) GET /api/profile/:profileId/other_profiles_recommendations
    Purpose: Suggests content that the user's OTHER profiles (on the same account) have watched or
    liked, excluding anything the requesting profile has already watched or liked itself.

    req.params =
    {
      profileId: string
    }

    res =
    {
      success: boolean,
      message: string,
      content:
      [
        {
          id: string,
          title: string,
          description: string,
          cover_image_name: string,
          type: "movie" | "series",
          categories: string[],
          release_date: string,
          age_limit: number,
          likes: number,
          createdAt: string,
          imdb_rating: number
        },
        ...
      ]
    }

17) GET /api/profile/:profileId/top_picks
    Purpose: Suggests up to 3 top-rated content items based on this profile's own taste -
    groups the categories of everything it has watched or liked, takes its top 3 most
    common categories, then returns the highest-rated other content sharing those
    categories (excluding anything already watched or liked).

    req.params =
    {
      profileId: string
    }

    res =
    {
      success: boolean,
      message: string,
      content:
      [
        {
          id: string,
          title: string,
          description: string,
          cover_image_name: string,
          type: "movie" | "series",
          categories: string[],
          release_date: string,
          age_limit: number,
          likes: number,
          createdAt: string,
          imdb_rating: number
        },
        ...
      ]
    }

    Note: returns an empty list (not an error) if the profile has no watch/like history yet.

CONTENT ENDPOINTS (/api/content)
--------------------------------------------------
("Content" means the movies/series in the system. Video files live on episodes, not on content
itself - every watchable item is an Episode, including a movie's single episode.)
(All content endpoints are public - no login needed.)

--------------------------------------------------
REVIEW ENDPOINTS (/api/reviews)
--------------------------------------------------
(A review is one profile's rating (1-10) plus an optional comment, for one specific episode.
A profile can have at most one review per episode.)

24) POST /api/reviews/:profileId/:contentId/:episodeId
    Purpose: Adds a review (rating + optional comment) for a specific episode, written by this
    profile. A profile can only review a given episode once. Adding a review automatically
    recalculates that content's average_rating and review_count.

    req.headers =
    {
      Authorization: "Bearer <token>"
    }

    req.params =
    {
      profileId: string,
      contentId: string,
      episodeId: string
    }

    req.body =
    {
      rating: number,
      comment?: string
    }

    res =
    {
      success: boolean,
      message: string,
      review?:
      {
        id: string,
        contentId: string,
        episodeId: string,
        profileId: string,
        rating: number,
        comment: string
      }
    }

    Note: rating must be a number between 1 and 10. comment has a 500 character limit.

25) PUT /api/reviews/:profileId/:contentId/:episodeId
    Purpose: Edits this profile's own review for a specific episode (rating and/or comment).
    Editing the rating automatically recalculates that content's average_rating.

    req.headers =
    {
      Authorization: "Bearer <token>"
    }

    req.params =
    {
      profileId: string,
      contentId: string,
      episodeId: string
    }

    req.body =
    {
      rating?: number,
      comment?: string
    }

    res =
    {
      success: boolean,
      message: string,
      review?:
      {
        id: string,
        contentId: string,
        episodeId: string,
        profileId: string,
        rating: number,
        comment: string
      }
    }

26) DELETE /api/reviews/:profileId/:contentId/:episodeId
    Purpose: Removes this profile's own review for a specific episode. Removing the review
    automatically recalculates that content's average_rating and review_count.

    req.headers =
    {
      Authorization: "Bearer <token>"
    }

    req.params =
    {
      profileId: string,
      contentId: string,
      episodeId: string
    }

    res =
    {
      success: boolean,
      message: string
    }

27) GET /api/reviews/
    Purpose: Searches/browses all reviews using query filters. Public, no login needed.

    req.query =
    {
      content_id?: string,
      episode_id?: string,
      profile_id?: string,
      user_id?: string,
      rating?: number,
      min_rating?: number,
      max_rating?: number,
      comment_starts?: string,
      comment_ends?: string,
      comment_contains?: string,
      limit?: number,
      skip?: number,
      sort?: string,
      sortOrder?: "greater_to_smaller" | "smaller_to_greater"
    }

    res =
    {
      success: boolean,
      message: string,
      reviews:
      [
        { id: string, contentId: string, episodeId: string, profileId: string, rating: number, comment: string },
        ...
      ]
    }

    Note: filter by episode_id to see all reviews of one episode, or by profile_id (together
    with episode_id) to find one profile's review of one episode.

--------------------------------------------------
GENERAL NOTES
--------------------------------------------------
- Every res always has a "success" field (true or false). Most also have a "message" field,
  except a handful of simple admin actions (tokens_count, kick, ban, ban-check) which only
  return "success" plus their specific data field.
- "?" after a field name means it is optional.
- All admin-only actions now live under /api/admin, consolidated by resource type
  (users, content, reviews) rather than scattered across each resource's own base path.
- req.headers = the request headers (used for the auth token).
- req.params = values that are part of the URL path (like :user_id).
- req.query = values after the "?" in the URL (like ?limit=10).
- req.body = the JSON object sent in the request body.
- sortOrder uses the same two values everywhere: "greater_to_smaller" (descending, default) or "smaller_to_greater" (ascending). Any other value returns an error.
- videoUrl now lives only on episodes, never on content itself.
- Reviews: rating is 1-10, comment is optional with a 500 character limit. A profile can only
  have one review per episode - adding a second is rejected, not merged or overwritten.
