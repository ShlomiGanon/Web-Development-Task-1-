API ENDPOINTS - SIMPLE OVERVIEW
================================
Base URL: /api
Auth: Send token in header -> Authorization: Bearer <token>

--------------------------------------------------
USER ENDPOINTS (/api/user)
--------------------------------------------------

1) POST /api/user/register
   Purpose: Creates a brand new user account.
   req.body = { email: string, phone: string, password: string, fullName: string, birthday: string }
   res = { success: boolean, message: string }

2) POST /api/user/login
   Purpose: Logs a user in and gives them a token to use for future requests.
   req.body = { email_or_phone: string, password: string }
   res = { success: boolean, message: string, token: string }

3) POST /api/user/logout
   Purpose: Logs the user out by cancelling their current token.
   req.headers = { Authorization: "Bearer <token>" }
   req.body = {}   (nothing needed)
   res = { success: boolean, message: string }

4) GET /api/user/me
   Purpose: Gets the profile info of the user who is currently logged in.
   req.headers = { Authorization: "Bearer <token>" }
   res = { success: boolean, message: string, user: { id, email, phone, fullName, birthday, createdAt } }

5) PUT /api/user/me
   Purpose: Lets the logged-in user update their own account details.
   req.headers = { Authorization: "Bearer <token>" }
   req.body = { password?: string, email?: string, phone?: string, fullName?: string, birthday?: string }
   res = { success: boolean, message: string, user: { id, email, phone, fullName, birthday, createdAt } }

6) GET /api/user/  (admin only)
   Purpose: Lets an admin search/browse through the list of all users.
   req.headers = { Authorization: "Bearer <token>" }
   req.query = { email_contains?, phone_contains?, fullname_contains?, born_after?, born_before?, joined_after?, joined_before?, limit?, skip?, sort?, sortOrder?: "greater_to_smaller"|"smaller_to_greater" }
   res = { success: boolean, message: string, users: [ {id, email, phone, fullName, birthday, createdAt}, ... ] }

7) GET /api/user/:user_id  (admin only)
   Purpose: Lets an admin view the details of one specific user.
   req.headers = { Authorization: "Bearer <token>" }
   req.params = { user_id: string }
   res = { success: boolean, message: string, user: { id, email, phone, fullName, birthday, createdAt } }

8) PUT /api/user/:user_id  (admin only)
   Purpose: Lets an admin update the account details of a specific user.
   req.headers = { Authorization: "Bearer <token>" }
   req.params = { user_id: string }
   req.body = { password?: string, email?: string, phone?: string, fullName?: string, birthday?: string }
   res = { success: boolean, message: string, user: { id, email, phone, fullName, birthday, createdAt } }

9) DELETE /api/user/:user_id  (super admin only)
   Purpose: Permanently deletes a user account.
   req.headers = { Authorization: "Bearer <token>" }
   req.params = { user_id: string }
   res = { success: boolean, message: string }

10) PUT /api/user/:user_id/permission  (admin only)
    Purpose: Promotes or demotes a user's permission level.
    req.headers = { Authorization: "Bearer <token>" }
    req.params = { user_id: string }
    req.body = { permission_level: number }
    res = { success: boolean, message: string }

--------------------------------------------------
PROFILE ENDPOINTS (/api/profile)
--------------------------------------------------
(All profile endpoints need: req.headers = { Authorization: "Bearer <token>" })
(A "profile" is like a Netflix-style profile - one user account can have several profiles)

11) POST /api/profile/
    Purpose: Creates a new profile under the logged-in user's account (max 4 profiles).
    req.body = {}   (nothing needed)
    res = { success: boolean, message: string, profiles: [ {id, profileName, age, ImageName}, ... ] }

12) DELETE /api/profile/:profileId
    Purpose: Deletes one of the user's profiles (must always keep at least one).
    req.params = { profileId: string }
    res = { success: boolean, message: string, profiles: [ {id, profileName, age, ImageName}, ... ] }

13) PUT /api/profile/:profileId
    Purpose: Updates the details of one profile (name, age, or picture).
    req.params = { profileId: string }
    req.body = { profileName?: string, age?: number, ImageName?: string }
    res = { success: boolean, message: string, profiles: [ {id, profileName, age, ImageName}, ... ] }

14) PUT /api/profile/
    Purpose: Updates several profiles belonging to the user in one request.
    req.body = { updates: [ { profileId: string, profileName?: string, age?: number, ImageName?: string }, ... ] }
    res = { success: boolean, message: string, profiles: [ {id, profileName, age, ImageName}, ... ] }

15) GET /api/profile/
    Purpose: Gets the full list of all profiles that belong to the logged-in user.
    res = { success: boolean, message: string, profiles: [ {id, profileName, age, ImageName}, ... ] }

16) GET /api/profile/:profileId
    Purpose: Gets the basic details of one specific profile.
    req.params = { profileId: string }
    res = { success: boolean, message: string, profile: {id, profileName, age, ImageName} }

17) GET /api/profile/:profileId/details
    Purpose: Gets the complete details of one profile, including watch history and liked content.
    req.params = { profileId: string }
    res = { success: boolean, profile: { id, profileName, age, ImageName, Liked_Media_IDs, LastWatched_Media_IDs, User_ID, createdAt, updatedAt } }

18) POST /api/profile/:profileId/likes/:contentId
    Purpose: Adds or removes a "like" on a specific movie/show for a specific profile (toggle).
    req.params = { profileId: string, contentId: string }
    req.body = {}   (nothing needed)
    res = { success: boolean, message: string, liked: boolean, likedMediaIds: [ contentId, ... ] }

19) POST /api/profile/:profileId/watch/:contentId
    Purpose: Records that a profile just watched a specific movie/show.
    req.params = { profileId: string, contentId: string }
    req.body = {}   (nothing needed)
    res = { success: boolean, message: string, watchHistory: [ contentId, ... ] }

--------------------------------------------------
CONTENT ENDPOINTS (/api/content)
--------------------------------------------------
("Content" means the movies/series in the system)

20) POST /api/content/  (admin only)
    Purpose: Adds a brand new movie or series to the catalog.
    req.headers = { Authorization: "Bearer <token>" }
    req.body = { title: string, type: "movie"|"series", release_date: string, description?: string, cover_image_name?: string, categories?: string[], age_limit?: number, videoUrl?: string }
    res = { success: boolean, message: string, content: { id, title, description, cover_image_name, type, categories, release_date, age_limit, likes, videoUrl, createdAt } }

21) DELETE /api/content/:contentId  (admin only)
    Purpose: Removes a movie/series from the catalog completely.
    req.headers = { Authorization: "Bearer <token>" }
    req.params = { contentId: string }
    res = { success: boolean, message: string }

22) PUT /api/content/:contentId  (admin only)
    Purpose: Edits the details of an existing movie/series.
    req.headers = { Authorization: "Bearer <token>" }
    req.params = { contentId: string }
    req.body = { title?: string, type?: "movie"|"series", release_date?: string, description?: string, cover_image_name?: string, categories?: string[], age_limit?: number, videoUrl?: string }
    res = { success: boolean, message: string, content: { id, title, description, cover_image_name, type, categories, release_date, age_limit, likes, videoUrl, createdAt } }

23) GET /api/content/:contentId
    Purpose: Gets the details of one specific movie/series. Public, no login needed.
    req.params = { contentId: string }
    res = { success: boolean, message: string, content: { id, title, description, cover_image_name, type, categories, release_date, age_limit, likes, videoUrl, createdAt } }

24) GET /api/content/
    Purpose: Searches/browses the whole catalog using filters like title, category, or type. Public, no login needed.
    req.query = { title_contains?, exact_category?, contain_category?, exclude_category?, type?, released_after?, released_before?, min_age_limit?, max_age_limit?, min_likes?, limit?, skip?, sort?, sortOrder?: "greater_to_smaller"|"smaller_to_greater" }
    res = { success: boolean, message: string, content: [ {id, title, description, cover_image_name, type, categories, release_date, age_limit, likes, videoUrl, createdAt}, ... ] }

--------------------------------------------------
GENERAL NOTES
--------------------------------------------------
- Every res always has a "success" field (true or false) and a "message" field.
- "?" after a field name means it is optional.
- req.headers = the request headers (used for the auth token).
- req.params = values that are part of the URL path (like :user_id).
- req.query = values after the "?" in the URL (like ?limit=10).
- req.body = the JSON object sent in the request body.
- sortOrder now uses the same two values everywhere: "greater_to_smaller" (descending, default) or "smaller_to_greater" (ascending). Any other value returns an error.
